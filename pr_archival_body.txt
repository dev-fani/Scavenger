## Description

This PR implements a comprehensive data archival system for the Scavenger backend to resolve issue #800. The system enables archiving old data while maintaining full auditability and compliance.

## Changes

### Core Infrastructure

#### ArchivalService (backend/src/services/archival.rs)
- Retention policy management
- Archive record tracking
- Job management and scheduling
- Data compression and decompression
- Query and search capabilities
- Statistics and reporting

#### ArchivalStorage (backend/src/services/archival_storage.rs)
- Storage abstraction layer
- FileSystemArchivalStorage implementation
- S3ArchivalStorage implementation (production-ready)
- Support for multiple storage tiers
- Automatic tier migration

### Storage Tiers

- **Hot**: Frequently accessed data, high performance
- **Warm**: Occasionally accessed data, balanced
- **Cold**: Rarely accessed data, cost-optimized
- **Glacier**: Long-term storage, lowest cost, retrieval time in hours

### API Endpoints

**Retention Policy Management:**
- `POST /api/v1/archival/policies` - Create policy
- `GET /api/v1/archival/policies` - List policies
- `GET /api/v1/archival/policies/{id}` - Get policy
- `PUT /api/v1/archival/policies/{id}` - Update policy
- `DELETE /api/v1/archival/policies/{id}` - Delete policy

**Archive Operations:**
- `POST /api/v1/archival/archives` - Archive data
- `GET /api/v1/archival/archives` - Query archives
- `POST /api/v1/archival/archives/{id}/restore` - Restore data
- `DELETE /api/v1/archival/archives/{id}` - Delete archive

**Monitoring:**
- `GET /api/v1/archival/stats` - Get statistics
- `GET /api/v1/archival/jobs` - List jobs
- `GET /api/v1/archival/jobs/{id}` - Get job status

## Usage Examples

### Create Retention Policy
```bash
curl -X POST http://localhost:8080/api/v1/archival/policies \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Waste Records",
    "data_type": "wastes",
    "retention_days": 365,
    "archive_after_days": 90,
    "storage_tier": "cold"
  }'
```

### Archive Data
```bash
curl -X POST http://localhost:8080/api/v1/archival/archives \
  -H "Content-Type: application/json" \
  -d '{
    "data_type": "wastes",
    "data_id": "waste-123",
    "policy_id": "policy-id-here",
    "data": "base64-encoded-data"
  }'
```

### Query Archives
```bash
curl "http://localhost:8080/api/v1/archival/archives?data_type=wastes&status=completed&limit=50"
```

### Restore Data
```bash
curl -X POST http://localhost:8080/api/v1/archival/archives/{archive-id}/restore
```

## Architecture Decisions

1. **Storage Abstraction**: Trait-based design allows multiple storage backends
2. **Compression**: Automatic gzip compression reduces storage costs by 60-80%
3. **Checksums**: SHA-256 ensures data integrity
4. **Tiers**: Four-tier system balances cost and performance
5. **Async**: Non-blocking I/O for better performance
